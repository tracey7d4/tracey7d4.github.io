Data structure - Graph

Undirected and Directed Graph

Posted on 2021, Apr 21 by Tracey Hoang 5 mins read
Table of Contents
    • Problem Description
    • Implementation

Source code

Graph is a data structure that represents pairwise relationships between vertices. It has two basic components - vertices and edges. The edges are used to connect pairs of vertices.

A graph can be directed or undirected. In an undirected graph, all the edges are bi-directional. It means that in an undirected graph, you can traverse in both directions from a vertex to the other one as long as there is an edge between them. Directed graph, on the other hand, has edges point in a single direction, and you can traverse in the given direction only.

A path in a graph \( G \) is a sequence of vertices \( v_0, v_1, ..., v_n \) so that for every \( i, 0 \leq i \leq n-1 \), \( (v_i, v_{i+1}) \) is an edge of \( G \).

Graph is used in a lot of real-world applications. For example, in Facebook, graph is used to represent the social connections among users and their friends, and Google map uses graph to build its map and graph’s path optimisation algorithms for their navigation system.

Let’s go though an small example as an illustration of graph in Go.

Problem Description

You are given a maze represented by an undirected graph, and you would like to check whether there is a path from given cell u to a given exit v from a maze.

Input format:

  • The first line contains 2 integers n and m - number of vertices and number of edges, respectively.
  • The next m lines are edges given by pairs of vertices.
  • The last line contains vertices u and v that we need to check whether or not the path between them.

Output: print YES if there is such a path, and NO otherwise.

Implementation

Create new type vertex to represent a vertex, and a graph is an array of vertices. A vertex is identified by its index in the array (graph) rather than holding an extra field (such as key field) in the struct.

A such vertex type contains 2 fields:

  • adjacent field contains a list of its adjacent that are stored in a map with keys are indices of adjacent, and values are pointers of those vertices;
  • flag field is a boolean value used to keep track of vertices that have been found. Its default value is false.
type vertex struct {
	adjacent   map[int]*vertex
	flag       bool
}

func newVertex() *vertex{
	return &vertex{
		make(map[int]*vertex),
		false,
	}
}

In this post I’ll represent the block of code used to read the input from standard input (Stdin).

func main() {
	scanner := bufio.NewScanner(os.Stdin)
	const maxCapacity = 1024*1024
	buf := make([]byte, maxCapacity)
	scanner.Buffer(buf, maxCapacity)
  // scan the first line of Stdin to get number of vertices n and number of edges m
	scanner.Scan()
	s := strings.Split(scanner.Text(), " ")
	n, _ := strconv.ParseInt(s[0], 10, 64) //number of vertices
	m, _ := strconv.ParseInt(s[1], 10, 64) // number of edges

You can increase the maxCapacity to read a larger input file.

As mentioned before, a graph is an array of given vertices, and is created as follows.

	graph := make([]*vertex, n) //graph is an array of vertices
	for i := 0; i < int(n); i++ {
		graph[i] = newVertex()
	}

Scan through the next m lines of input file for given edges information. As it is an undirected graph, a vertex u is an adjacent of v, and vice versa, v is an adjacent of u.

	for i := 0; i < int(m); i++ {
		scanner.Scan()
		data := strings.Split(scanner.Text(), " ")
		u, _ := strconv.ParseInt(data[0], 10, 64)
		v, _ := strconv.ParseInt(data[1], 10, 64)
		graph[u-1].adjacent[int(v)-1] = graph[v-1]
		graph[v-1].adjacent[int(u)-1] = graph[u-1]
	}

Now scan the last line of input for the pair of vertices u and v that we need to see is v reachable from u.

  scanner.Scan()
  data := strings.Split(scanner.Text(), " ")
  u, _ := strconv.ParseInt(data[0], 10, 64)
  v, _ := strconv.ParseInt(data[1], 10, 64)
  isConnected(graph, int(u)-1, int(v)-1)
}

where isConnected() is a function used to check whether the given pair of vertices is connected, and is defined as follows

func isConnected(node []*vertex, j, k int) {
	q := newQueue()
	q.push(node[j])
	node[j].flag = true
	for !q.isEmpty() {
		s := q.pop()
		for key, value := range s.adjacent {
			if key == k {
				fmt.Println("YES")
				return
			}
			if !value.flag {
				value.flag = true
				q.push(value)
			}
		}
	}
	fmt.Println("NO")
}

In order to check whether vertex v is reachable from u, we need to explore every edge leaving every vertex we have found. If vertex v is found on the way of exploring, there definitely is a path between them, so output YES and stop the discovering process. Otherwise, keep exploring the edges until no more edges to be explored, then output NO.

A queue is used to store a list of visited vertices. When a vertex is visited, its flag field is set to true, and is pushed to the queue. I explore new edges widely, or in breath-first order.

Some of follows is supporting functions.

type queue []*vertex
func newQueue() *queue{
	return &queue{}
}
func (q *queue) isEmpty() bool {
	return len(*q) == 0
}
func (q *queue) push(v *vertex) {
	*q = append(*q,v)
}
func (q * queue) pop() *vertex {
	res := (*q)[0]
	*q = (*q)[1:]
	return res
}

Let’s run our program with the following sets of input:

4 4
1 2
3 2
4 3
1 4
1 4

There obviously is a path between vertex 1 and 4, so the output should be YES.

4 2
1 2
3 2
1 4

Vertex 1 and 4 are not connected, so the output should be NO.
